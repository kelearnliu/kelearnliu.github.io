文档中列出的是根据自己的思考，再结合代码随想录和LeetCode官网的答案，自己作出的总结。同时在解题过程中会有一些自己的想法和辅助函数都一并列出，LeetCode对应题目链接在各题目处给出，完整的解题代码可访问本人的GitHub得到.

相比于贪心算法，动态规划算法是要考虑状态转移的，不能只选取局部最优值。

一般分为四个步骤：

  1. 确定$dp$数组以及下标的含义
  2. 确定递推公式
  3. $dp$数组如何初始化
  4. 确定遍历顺序
  5. 举例推导$dp$数组

1.斐波那契数列

斐波那契数 （通常用 F(n) 表示）形成的序列称为 斐波那契数列 。该数列由 0 和 1 开始，后面的每一项数字都是前面两项数字的和。也就是：

动态规划算法中，使用的是前两项的和，这样的话可以先保存一下两项，不断迭代就能计算出下一项的数值。

还可以通过状态转移矩阵得到递推关系，然后对状态转移矩阵求n次方，就能够得到任意两项和$F(0),F(1)$之间的关系了。

  • 时间复杂度：$O(n)$，遍历一遍
  • 空间复杂度：$O(n)/O(1)$ 不要全存储即可是常数空间

2.爬楼梯

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

由于当前状态只与前两个状态有关，那么就只要考虑前面两步即可。$dp[i]=dp[i-1]+dp[i-2]$ ,这里存储的是到达第i级的情况数目。

可以创建一个和n一样大的数组存储每一种情况下的方法，也可以值保存两个变量来更新当前状态的方法数，可以节省空间，从$O(n)->O(1)$

时间复杂度：$O(n)$，遍历一遍

空间复杂度：$O(n)/O(1)$ 不要全存储即可是常数空间

3.使用最小花费爬楼梯

你可以选择从下标为 0 或下标为 1 的台阶开始爬楼梯。

请你计算并返回达到楼梯顶部的最低花费。

这里对于$dp$数组的设置是n+1个阶梯，这里的0级阶梯是没有含义的，所以初始化时要注意数组长度的选择。

时间复杂度：$O(n)$，遍历一遍

空间复杂度：$O(n)/O(1)$ 不要全存储即可是常数空间

7.不同的二叉搜索树

给你一个整数 n ，求恰由 n 个节点组成且节点值从 1 到 n 互不相同的 二叉搜索树 有多少种？返回满足题意的二叉搜索树的种数。

思想是将节点进行左右分枝，左子树节点数量+右子树节点数量=i-1;然后从0:i-1进行遍历，将所有的情况相加就能得到最后的种类。

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$O(n)$

8.背包理论I

几个不同重量的物品，每个都有价值，给定一个背包，里面装的物品总价值最高。

重点1：$dp$数组的含义，$dp[i][j]$代表从物品$0-i$里面选择，装入容量为$j$的背包时的最大值

重点2：初始化边界值，第一行和第一列都要初始化

重点3：状态的转移

情况1：从$dp[i-1][j]$转移到$dp[i][j]$时，如果$j$在$i-1$装入以后还有空余空间，那么就要判断剩余的空间是否能够放下第$i$个，如果不行就保持原来的值

情况2：如果能放下，就要比较

当前$dp[i-1][j]$加上第$i$个物品的价值和$j-weight[i]$那个背包+扩容到j时的价值，取最大的作为返回值。

  • 时间复杂度：$O(nm)$， 是物品数背包容量值
  • 空间复杂度：$O(n*m)$

9.背包理论II

对于上面的问题，可以使用一维动态数组进行表示。这里的第二种方法就使用滚动$dp$数组来进行计算。

这里的变化在于

$dp[i][j]=max(dp[i-1][j],dp[i-1][j-weight[i]]+value[i]);$

变为

$dp[j]=max(dp[j],dp[j-weight[i]]+value[i]);$

这里的$dp[j]$代表着容量为$j$的背包最大的价值，相当于把上一层数组拷贝下来了。

这里面的遍历物品和遍历背包时有顺序的，先要遍历物品，再倒序遍历背包。

可以理解成用不同的物品装进容量不同的背包中去，从小到大往里面装，然后更新不同容量时的价值,如果加入的价值没有之前的价值高，就不作更新，保证了dp数组的最优。所以也可以从最后一个物品向前遍历。从后向前的原因是，上一次的数组已经保留了相关的数据，下一次遍历从后向前就意味着使用前面的结果，与二维dp等价。先遍历物品再遍历背包，如果先遍历背包再遍历物品那就是 只装一个物品进这个背包了。

  • 时间复杂度：$O(nm)$， 是物品数背包容量值
  • 空间复杂度：$O(m)$

10.分割等和子集

给你一个 只包含正整数 的 非空 数组 nums 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

本体可以使用上一章的回溯算法解决，但是时间复杂度很高，因此使用DP算法进行解决。

这里需要对问题进行转换，经典的背包问题中的，每一个物品有两个属性，$weight,value$，这里的数组中$weight=value$，那么容量为$j$的最大价值也就转换成了最大重量。

$dp[j]$代表容量为$j$,容量为$dp[j]$

转移方程：$dp[j]=max(dp[j], dp[j-weight[i]]+value[i])$

由于这里$weight=value$，所以得到的是

$dp[j]=max(dp[j], dp[j-nums[i]]+nums[i])$

这里的判断条件式：$dp[target]==target$ 如果两个相等，那么就说明刚好装满。

  • 时间复杂度：$O(n^2)$，相比于回溯算法的$O(2^n)$降低了很多
  • 空间复杂度：$O(n)$

11.最后一块石头的重量II

有一堆石头，用整数数组 stones 表示。其中 stones[i] 表示第 i 块石头的重量。

每一回合，从中选出任意两块石头，然后将它们一起粉碎。假设石头的重量分别为 x 和 y，且 x <= y。那么粉碎的可能结果如下：

  • 如果 x == y，那么两块石头都会被完全粉碎；
  • 如果 x != y，那么重量为 x 的石头将会完全粉碎，而重量为 y 的石头新重量为 y-x。

最后，最多只会剩下一块 石头。返回此石头 最小的可能重量 。如果没有石头剩下，就返回 0。

要将石头分为尽可能重量相近的两堆石头，这样能够保证最后剩下的是最小的。其实也就是像上一道题一样，分成两堆相同的，那剩下的质量就是我们要得到的最后一块石头的重量。

这里使用的accumulate函数可以使用lambda表达式，这类似于使用sort表达式中的lambda表达式来完成比较大小或者累加的任务。这个lambda表达式是在第四个参数的位置，第三个参数是累加的初值

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$O(n)$，和上一道题一样，数组的规模可能会很大

12.目标和

给你一个整数数组 nums 和一个整数 target 。

向数组中的每个整数前添加 '+' 或 '-' ，然后串联起所有整数，可以构造一个 表达式 ：

  • 例如，nums = [2, 1] ，可以在 2 之前添加 '+' ，在 1 之前添加 '-' ，然后串联起来得到表达式 "+2-1" 。

返回可以通过上述方法构造的、运算结果等于 target 的不同 表达式 的数目。

还是问题转换：就是把$nums$数组中的数字分为两类，一类作为正数的集合，一类作为负数的集合，两个集合各自的和相减就是$target$，那么就可以转化成0-1背包问题，也就是上面一道题的思想。

$\sum{left}+\sum{right}=sum$

$\sum{left}-\sum{right}=target$

就可以知道

$\sum{left} = (sum+target)/2$

如果右边不是正数，说明不能得到题设的target，则此题无解。

$dp$数组的含义：容量为$j$的时候，有$dp[j]$种方法

如果物体为1，那么可以使用$dp[4]$来凑成$dp[5]$

如果物体为2，那么可以使用$dp[3]$来凑成$dp[5]$

这里物体2不会使$dp[3]变为dp[3]+1$，因为给$dp[3]$中的每个方法加二就行，结果还是$dp[3]$。

所以递推公式是$dp[j]+=dp[j-num[i]]$ 装满背包的方法用这个公式

这里的初始化要注意：$dp[0]=1，dp[n!=0]=0$

因为$dp[0]$是在公式中一切递推结果的起源，如果$dp[0]$是0的话，递推结果将都是0。

  • 时间复杂度：$O(n × m)$，n为正数个数，m为背包容量
  • 空间复杂度：$O(m)$，m为背包容量

13.一和零

给你一个二进制字符串数组 strs 和两个整数 m 和 n 。

请你找出并返回 strs 的最大子集的长度，该子集中 最多 有 m 个 0 和 n 个 1 。

如果 x 的所有元素也是 y 的元素，集合 x 是集合 y 的 子集 。

也就是装满一个背包，最多能放多少个物品。

$dp[i][j]$代表$i个0和j个1$，最多背$dp[i][j]$个物品

$dp[j]=max(dp[j], dp[j-weight[i]]+value[i])$

就变为

$dp[i][j]=max(dp[i][j], dp[i-x][j-y]+1)$

思想是一样的，后面项的意思是当前物品中有$x个0和y个1$，当放入这个物品时，必须找到能够存放这个物品的那个最大的背包，然后+1就是把这个物品装进去。

  • 时间复杂度：$O(n*m^3)$ $n个字符串，一个字符串长度为m,0有k个，1有(m-k)个$
  • 空间复杂度：$O(n^2)$，和上一道题一样，数组的规模可能会很大

15.零钱兑换II

给你一个整数数组 coins 表示不同面额的硬币，另给一个整数 amount 表示总金额。

请你计算并返回可以凑成总金额的硬币组合数。如果任何硬币组合都无法凑出总金额，返回 0 。

假设每一种面额的硬币有无限个。

题目数据保证结果符合 32 位带符号整数。

这道题和13.目标和问题很相似，就是求元素的和等于一个定值，这里对于元素的使用数量不做限制。

本题中有一个问题需要搞清楚，就是求的是组合，这意味着顺序不同的一组值对应的是同一个组合，这里区别就是排列与组合的区别。排列顺序不同的一组值是不同的，组合就是相同的。

这里区别排列与组合的点在于遍历顺序，这里要求给定数组中是没有相同元素的。

  • 排列：先遍历背包容量，再遍历物品
  • 组合：先遍历物品，再遍历背包容量，不会出现物品顺序不同的结果

确定$dp$数组的含义：$dp[j]$代表着背包容量为$j$时，有$dp[j]$种组合方式。

$dp[0]=1, dp[n!=0]=0$ ，背包为0的情况下是1，这个是系统默认的值，可以理解成有一种，这种情况是不存在？？

递推公式和12.目标和的是一样的

$dp[j] += dp[j-nums[i]]$

  • 时间复杂度：$O(n*amount)$ $n个元素，背包容量为amount$
  • 空间复杂度：$O(n)$，和上一道题一样，数组的规模可能会很大

16.组和总和IV

给你一个由 不同 整数组成的数组 nums ，和一个目标整数 target 。请你从 nums 中找出并返回总和为 target 的元素组合的个数。

题目数据保证答案符合 32 位整数范围。

这个题是排列的问题，遍历的方法即使先遍历背包再遍历物品。

这样才能保证有不同顺序物品加入背包中，如果先遍历背包，那物品就必须是按照顺序加入的，不会存在不同顺序的情况。

排列问题进行动态规划时，需要考虑物品的下标是否小于0，放着访问越界。上面组合问题不考虑的原因在于后遍历背包时的倒序遍历能保后面状态转移方程中的序号不为0。

  • 时间复杂度：$O(n*amount)$ $n个元素，背包容量为amount$
  • 空间复杂度：$O(n)$，和上一道题一样，数组的规模可能会很大

18.零钱兑换

给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。

计算并返回可以凑成总金额所需的 最少的硬币个数 。如果没有任何一种硬币组合能组成总金额，返回 -1 。

你可以认为每种硬币的数量是无限的。

$dp$数组的含义：容量为$j$的时候，最少得硬币数量是$dp[j]$。

这里和15.零钱兑换的区别在于，前者的递推公式中，求得的的组合的方法，这里求的是组合中硬币的个数，所以转移方程不同，方法不需要+1，硬币个数需要+1。

$dp[j] = min(dp[j-nums[i]]+1, dp[j])$

$dp[0]=0,dp[n!=0]=INT_MAX$

  • 时间复杂度：$O(n*m)$ $n个元素，背包容量为m$
  • 空间复杂度：$O(1)$

22.打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

对于当前的第$i$个房屋，主要的策略是考虑上一个和上两个房屋的情况。

$dp[i-2]+num[i] / dp[i-1]$

如果上一个屋子不打劫，那么就是左边一项所表示的；如果上一个屋子打劫，就沿用$dp[i-1]$，然后在这两个中取最大值来决定要不要打劫。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(n)$

24.打家劫舍III

在二叉树结构中，不能偷相连的两个节点的屋子。

每个节点都有一个$dp$的二维数组来存储是否选择这个节点，0代表选择，1代表不选择。那么递归的时候如果选择此节点，就得到$val1$，不选的话$val2$就是不选择此节点的结果。然后取两种情况的最大值。

  • 时间复杂度：$O(n)$，每个节点只遍历了一次
  • 空间复杂度：$O(log n)$，算上递推系统栈的空间

25.买卖股票的最佳时机

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

这里可以使用贪心算法，用$low$表示目前最小的价格，然后在遍历过程中寻找最大的。

  • 时间复杂度：$O(n)$，每个节点只遍历了一次
  • 空间复杂度：$O(1)$

还可以使用动态规划，这里的$dp$数组是二维的，持有或者不持有。

$dp[i][0]持有股票的最大现金，dp[i][1]不持有股票的最大现金$

这两种情况能够把所有的情况囊括进来。

状态方程中：

第i天持有时的资金=max(前一天持有时的资金, 今天买入）

第i天不持有时的资金=max(前一天不持有时的资金, 昨天不持有+今天买入）
$$
dp[i][0]=max(dp[i-1][0], -price[i])
$$

$$
dp[i][1]=max(dp[i][1],dp[i-1][0]+price[i])
$$

优化的地方在于可以将数组缩为2*2个，因为只和前面的两个状态有关。实现的方法是对2取余，这样就能保证在原来的dp数组上更新了。

  • 时间复杂度：$O(n)$，每个节点只遍历了一次
  • 空间复杂度：$O(n)$

26.买卖股票的最佳时机II

这道题和贪心算法的那道题一样，只需要收集正的序列和即可。

贪心算法的思路就是收集正数项的个数，得到最大的正数和，这样就能够计算出最大的报酬。

对于DP算法，这个框架结构和第25题一样，但是区别在于本题的股票是可以买入卖出多次的，这就意味着对于$dp[i][0]$的更新需要调整。

上一题的$dp[i][0]$考虑的是保持之前持有状态和买入的最大值，这一题就要求要对于后面一项细化分析，因为买入之前必然已经卖出或者没买，就要加上上一次的$dp[i-1][1]$

  • 时间复杂度：$O(n)$，每个节点只遍历了一次
  • 空间复杂度：$O(n)$

27.买卖股票的最佳时机III

这道题的要求是只能买入卖出2次，求最大的收益。

这里扩展了状态，从持有和不持有两个状态 扩展成了 （不操作，第一次买入，第一次卖出，第二次买入，第二次卖出）。

主要是初始值的设置，可以使用当天买卖来确定初始化值

$dp[0][0]=0$

$dp[0][1]=-price[i]$

$dp[0][2]=0$

$dp[0][3]=-price[i]$

$dp[0][4]=0$

对于最后$dp[i][2]和dp[i][4]$的选择，一定选的是4，因为可以当天买入卖出，所以4一定包含2。

  • 时间复杂度：$O(n)$，每个节点只遍历了一次
  • 空间复杂度：$O(n)$，这里可以只维护一个一维dp，因为更新只是依靠于前一时刻的值，所以可以简化为$O(1)$

28.买卖股票的最佳时机IV

  • 时间复杂度：$O(n*k)$

29.买卖股票的最佳时机含冷冻期

本题要求卖出以后，必须等待一天才能再次买入。

这里对于$dp$数组分析是要将持有和不持有的两种情况进行再次地细化，因为多了一个冷冻期。不持有股票状态下，如果刚卖出股票，那么就不能够进入冷冻期，如果保持不持有状态，那么就能够进入冷冻期。

$dp[i][0]持股$

1和2就是不持有股票的细分

$dp[i][1]保持卖出状态$

$dp[i][2]本次卖出$

$dp[i][3]冷冻期$

保持上一次的情况，上一次冷冻期+这一次买入，上一次没买+这一次买入
$$
dp[i][0]=max(dp[i-1][0],dp[i-1][1]-price[i],dp[i-1][3]-price[i])
$$
保持上一次卖出的情况+冷冻期
$$
dp[i][1]=max(dp[i-1][1],dp[i-1][3])
$$
上一次卖出的话 这一次就是持有的股票+卖出的收益
$$
dp[i][2]=dp[i-1][0]+price[i]
$$
冷冻期的话 一定是上一次卖出了
$$
dp[i][3]=dp[i-1][2]
$$
初始化：

$dp[0][0]=-price[0]$

$dp[0][1]=0$ 这是从递推公式中反推的，要保证递推正确。

$dp[0][2]=0$当天买当天卖

$dp[0][3]=0$ 递推公式中来的。

对于非法状态的初始化，可以从递推公式反推，类似于两个函数要连续时，公共点的值。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(n)$

31. 最长递增子序列

给你一个整数数组 nums ，找到其中最长严格递增子序列的长度。

子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

$dp[i]$代表以$nums[i]$结尾的最长递增子序列的长度。

$设想j在i前，dp[i]的数值就是dp[j]+1和dp[i]的最大值$

所以状态转移方程就是
$$
dp[i]=max(dp[i],dp[j]+1)
$$

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$O(n)$

方法2：设置一个新的空数组，遍历数组的元素，并插入空序列中。插入的条件是，当当前元素不在新数组中时，就加入；如果使用二分查找找到此元素的值可以插入数组中(如2就可以加入{1,3})。方法是贪心+二分查找。

$lower_bound( begin,end,num)$：从数组的begin位置到end-1位置二分查找第一个大于或等于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

$upper_bound( begin,end,num)$：从数组的begin位置到end-1位置二分查找第一个大于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

  • 时间复杂度：$O(n*logn)$
  • 空间复杂度：$O(n)$

33. 最长重复子数组

给两个整数数组 nums1 和 nums2 ，返回 两个数组中 公共的 、长度最长的子数组的长度 。

$dp[i][j]$是以$i-1结尾的nums1$和$j-1结尾的nums2$的最长重合子数组。

这里定义不是$i,j为结尾的子数组$，这是为了方便初始化，相当于给左上角加了一行一列，只需要给他们全部初始化为0即可。

递推公式：两个数组的ij都要同时向后推。边界条件是0
$$
dp[i][j] = dp[i-1][j-1]+1;

dp[i][0]=dp[0][j]=0
$$

  • 时间复杂度：$O(n*m)$，$n$ 为A长度，$m$为B长度
  • 空间复杂度：$O(n)$

34.最长公共子序列

给定两个字符串 text1 和 text2，返回这两个字符串的最长 公共子序列 的长度。如果不存在 公共子序列 ，返回 0 。

一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

定义$dp$数组的方式和上一题类似

$以[0,i-1]的nums1和以[0,j-1]的nums2$的最长公共子序列的长度
$$
if(nums1[i-1]==nums2[j-1])
dp[i][j] = dp[i-1][j-1]+1;
else
mdp[i][j]=max(dp[i-1][j],dp[i][j-1])
dp[i][0]=dp[0][j]=0
$$
对于不相等时的理解，如果i-1和j-1不相等，那么就说明要去看$nums1[0:i-2]和nums2[0:j-1]$或者$nums2[0:i-2]和nums1[0:j-1]$的公共子序列并且取大。

这里和上一题的区别在于：上一题是重复，这一题是公共。所以要再else讨论。

  • 时间复杂度：$O(n*m)$，$n$ 为A长度，$m$为B长度
  • 空间复杂度：$O(n)$

35.不相交的线

在两条独立的水平线上按给定的顺序写下 nums1 和 nums2 中的整数。

现在，可以绘制一些连接两个数字 nums1[i] 和 nums2[j] 的直线，这些直线需要同时满足满足：

  • nums1[i] == nums2[j]
  • 且绘制的直线不与任何其他连线（非水平线）相交。

请注意，连线即使在端点也不能相交：每个数字只能属于一条连线。

以这种方法绘制线条，并返回可以绘制的最大连线数。

相当于去找最长的公共子序列，也就是上一道题34.最长公共子序列

  • 时间复杂度：$O(n*m)$，$n$ 为$nums1$长度，$nums2$为B长度
  • 空间复杂度：$O(n)$

36.最大子序列和（贪心算法第3题）

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组 是数组中的一个连续部分。

定义$dp$数组的含义是，以$nums[i]$结尾的数组所存放的最大的和。

状态转移方程为：要么在上一个基础上加一个 要么重新开始计算
$$
dp[i]=max(dp[i-1]+nums[i],nums[i])
$$
在每一次得到$dp[i]$的时候更新当前最大值
$$
if (dp[i] > result) result = dp[i];
$$

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(n)$

37.判断子序列

给定字符串 s 和 t ，判断 s 是否为 t 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，"ace"是"abcde"的一个子序列，而"aec"不是）。

这个题和34.最长公共子序列是相似的，这里多了一个判断条件，即最长公共子序列的长度应该和s的长度相同。

而且这里给定了要判断s是不是t的子序列，只需要考虑t中需要删除元素的情况。所以在递推的过程中，
$$
dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
$$

$$
要变为
dp[i][j] = dp[i][j - 1];
$$

  • 时间复杂度：$O(n*m)$，$n$ 为$s$长度，$t$为B长度
  • 空间复杂度：$O(n)$

39. 两个字符串的删除操作

给定两个单词 word1 和 word2 ，返回使得 word1 和 word2 相同所需的最小步数。

每步 可以删除任意一个字符串中的一个字符。

$dp[i][j]$:以i-1为结尾的字符串word1，和以j-1位结尾的字符串word2，想要达到相等，所需要删除元素的最少次数

当两个字符串的元素相等时，DP数组不发生变化，是因为不需要删除元素，所以数组的值不需要变化。

$if(w1[i-1]==w2[j-1])
dp[i][j] = dp[i-1][j-1];$

如果过两个元素不相等

$dp[i][j] = min({dp[i - 1][j - 1] + 2, dp[i - 1][j] + 1, dp[i][j - 1] + 1});$

初始化的方程是：含义是当一个字符串是0的时候，要删除i/j次

$dp[i][0] = i,dp[0][j] = j$

  • 时间复杂度：$O(n*m)$，$n$ 为$s$长度，$t$为B长度
  • 空间复杂度：$O(n*m)$

41. 回文子串

给你一个字符串 s ，请你统计并返回这个字符串中 回文子串 的数目。

回文字符串 是正着读和倒过来读一样的字符串。

子字符串 是字符串中的由连续字符组成的一个序列。

具有不同开始位置或结束位置的子串，即使是由相同的字符组成，也会被视作不同的子串。

动态规划思路：考虑两边两个元素的值是否相同，如果过相同，且包夹的字符串是回文，那么当前两个元素包夹的字符串也是回文。

对于包夹部分的字符串分类：1.只有一个或者2个，那一定是回文

2.多个的话 继续$dp$

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$dp O(n^2)$
  • 空间复杂度：$双指针 O(1)$

42.最长回文子序列

相对于上一题，不需要连续，只要存在就行

$if(s[i]==s[j])
dp[i][j] = dp[i+1][j-1]+2;$

$else dp[i][j] = max(dp[i][j-1],dp[i+1][j])$

在递推过程中，需要包夹部分的状态，最后就是$i==j$的时候的值最先要知道，所以要先初始化

$for(int i=0;i<size;i++) dp[i][i]=1$

最后的遍历顺序可以从递推公式中知道，是从左下开始遍历，所以i要从size-1开始递减。

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$O(n^2)$
