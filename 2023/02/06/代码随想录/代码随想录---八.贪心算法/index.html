第八章 贪心算法

[toc]
文档中列出的是根据自己的思考，再结合代码随想录和LeetCode官网的答案，自己作出的总结。同时在解题过程中会有一些自己的想法和辅助函数都一并列出，LeetCode对应题目链接在各题目处给出，完整的解题代码可访问本人的GitHub得到.

贪心算法并无实际套路，能够自圆其说就行。比如通过局部最优去找到全局最优。

一般分为四个步骤：

  1. 将问题分解为子问题
  2. 找出适合的贪心策略
  3. 求解每一个子问题的最优解
  4. 将局部最优解堆叠成全局最优解

自己的疑惑：

这个对于强凸的问题是成立的，但是对于一些问题可能不成立？

1. 分发饼干

假设你是一位很棒的家长，想要给你的孩子们一些小饼干。但是，每个孩子最多只能给一块饼干。

对每个孩子 i，都有一个胃口值 g[i]，这是能让孩子们满足胃口的饼干的最小尺寸；并且每块饼干 j，都有一个尺寸 s[j] 。如果 s[j] >= g[i]，我们可以将这个饼干 j 分配给孩子 i ，这个孩子会得到满足。你的目标是尽可能满足越多数量的孩子，并输出这个最大数值。

这个问题有点像最近看到的美团的笔试题，里面是袋子装巧克力

这个题里面是根据容量排序，再求前向的和，二分查找得到最后的结果。

时间复杂度：$O(mlogm+nlogn)$ ,其中m和n是g和s的长度，排序的时间复杂度就是$O(mlogm+nlogn)$ ，双指针遍历的时间复杂度是$O(m+n)$。

空间复杂度：$O(logm+logn)$，主要是排序的额外空间开销。

2. 摆动序列

如果连续数字之间的差严格地在正数和负数之间交替，则数字序列称为 摆动序列 。第一个差（如果存在的话）可能是正数或负数。仅有一个元素或者含两个不等元素的序列也视作摆动序列。

这个问题在于对于特殊情况的讨论

  • 峰值或者峰谷有平坡
  • 上升过程有平坡
  • 只有首尾元素的时候的计数问题

时间复杂度：$O(n)$, 遍历一遍；

空间复杂度：$O(1)$，只存放两个临时变量。

3.最大子序和

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

解题思想：维护一个result变量作为最终返回值，定义一个count变量作为当前序列和，当count>result的时候返回它。

这有一个误区，我刚开始也有一点陷进去，就是当遇到一个负数时，比如4，-1，这个-1会让总和减小，为什么还要把它加进去呢？

原因在于，我们在判断值的时候，在-1之前的这个子序列如果已经比当前存储的大，就已经作为result了，不会影响当前最大的总和的取值。

时间复杂度：$O(n)$, 遍历一遍；

空间复杂度：$O(1)$，只存放两个临时变量。

4. 买卖股票的最佳时机II

给你一个整数数组 prices ，其中 prices[i] 表示某支股票第 i 天的价格。在每一天，你可以决定是否购买和/或出售股票。你在任何时候 最多 只能持有 一股 股票。你也可以先购买，然后在 同一天 出售。返回 你能获得的 最大 利润 。

关键在于每一段时间之间差值的计算，主要思想是：如果在两天之间股票的增加值是幅值，那么就在当天将股票抛出。就相当于对原来的数组进行差分计算，得到每一天递增的值，将连续的正值作为一段收益，最后将所有的收益累加起来就可以了。

时间复杂度：$O(n)$, 遍历一遍；

空间复杂度：$O(1)$，只存放1个临时变量。

5. 跳跃游戏

给定一个非负整数数组 nums ，你最初位于数组的 第一个下标 。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个下标。

思想：维护一个cover变量，保存当前能够达到的最远的位置，如果cover>size，那么就说明能够达到最远的位置，返回true就行了。

时间复杂度：$O(n)$, 遍历一遍；

空间复杂度：$O(1)$，只存放1个临时变量。

6. 跳跃游戏II

给定一个长度为 n 的 0 索引整数数组 nums。初始位置为 nums[0]。

每个元素 nums[i] 表示从索引 i 向前跳转的最大长度。换句话说，如果你在 nums[i] 处，你可以跳转到任意 nums[i + j] 处:

  • 0 <= j <= nums[i]
  • i + j < n

返回到达 nums[n - 1] 的最小跳跃次数。生成的测试用例可以到达 nums[n - 1]。

这个题比上个题难的点在于：不仅要计算当前的覆盖范围，还要计算下一个覆盖范围，只有当覆盖范围在不断变大的时候，我们才选择走到这一个位置。

时间复杂度：$O(n)$, 遍历一遍；

空间复杂度：$O(1)$，只存放1个临时变量。

7. K次取反后最大化的数组和

给你一个整数数组 nums 和一个整数 k ，按以下方法修改该数组：

  • 选择某个下标 i 并将 nums[i] 替换为 -nums[i] 。

重复这个过程恰好 k 次。可以多次选择同一个下标 i 。

以这种方式修改数组后，返回数组 可能的最大和 。

这里的思想是，先尽可能将负数反转成正数，如果此时的k还是大于0的，就要考虑k的值。如果k对2取余的结果是0，说明翻转偶数次就可将k用完，这样的话原数组的和可以不变，如果是1，说明还要额外翻转一次，那么就要找数组中最小的正数去翻转才能保证和最大。

这也是为什么使用绝对值对数组进行排序的原因。

时间复杂度：$O(n*logn)$, 主要是排序的时间；

空间复杂度：$O(1)$，只存放1个临时变量。

8.加油站

在一条环路上有 n 个加油站，其中第 i 个加油站有汽油 gas[i] 升。

你有一辆油箱容量无限的的汽车，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 gas 和 cost ，如果你可以绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 保证 它是 唯一 的。

考虑两种解法：暴力解法和贪心算法

对于暴力解法，就是模拟一遍过程，要注意的是对于内部环的模拟，使用while来做效率比较高

对于贪心算法，就是从每个站点开始计算当前剩余的油料和，如这个油料和是小于0的，那么就说明从开始到当前站点是无法到达的。此时再次更新节点，就以当前和为负的最后一个节点的下一个为开始。（因为加上了最后一个节点的消耗，导致和为负，所以要以最后一个节点的下一个节点开始）。

暴力解法：

时间复杂度：$O(n^2)$

空间复杂度：$O(1)$

贪心解法：

时间复杂度：$O(n)$

空间复杂度：$O(1)$

9.分发糖果

n 个孩子站成一排。给你一个整数数组 ratings 表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

  • 每个孩子至少分配到 1 个糖果。
  • 相邻两个孩子评分更高的孩子会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的 最少糖果数目 。

从左往右比较一遍，再从右往左比较一遍，就能够得到结果。

但是对于LC中的第二种解法，还是不是很明白。

时间复杂度：$O(n)$，遍历了2遍数组。

空间复杂度：$O(n)$,储存了一个数组。

这个算法是更好一点的，但是我不太能理解。

时间复杂度：$O(n)$，遍历了2遍数组。

空间复杂度：$O(1)$， 维护了4个变量。

10.柠檬水找零

在柠檬水摊上，每一杯柠檬水的售价为 5 美元。顾客排队购买你的产品，（按账单 bills 支付的顺序）一次购买一杯。

每位顾客只买一杯柠檬水，然后向你付 5 美元、10 美元或 20 美元。你必须给每个顾客正确找零，也就是说净交易是每位顾客向你支付 5 美元。

注意，一开始你手头没有任何零钱。

给你一个整数数组 bills ，其中 bills[i] 是第 i 位顾客付的账。如果你能给每位顾客正确找零，返回 true ，否则返回 false 。

这里的售前方式只有三种5,10,20，所以情况很简单

  • 收5块，直接存下
  • 收10块，有5块余额则存下，否则返回false
  • 收20块，有1张10块+1张5块 或者3张5块 就收下 否则返回false

理清楚这三种情况就能够解决问题，我倾向于使用一个map来保存这种对应关系。

11.根据身高重建队列

假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。

请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。

当前有h和k两个维度，必须逐个考虑各个维度才能作出选择。这里选择身高作为第一次排序的依据，这样就能够再使用k再重新排序得到有序的数组。

这里使用一个list的类型来存储，这样的话比较节省时间。

如果用vector的话，使用insert插入需要的时间就是$O(n^2)$，而且过程中如果数组个数比原先给定的大时，还要扩容，这会将原来所有的元素都拷贝一遍，所以使用list更快，直接插如。

时间复杂度：$O(nlog n + n^2)$

空间复杂度：$O(n)$

12.用最少数量的箭引爆气球

有一些球形气球贴在一堵用 XY 平面表示的墙面上。墙面上的气球记录在整数数组 points ，其中points[i] = [xstart, xend] 表示水平直径在 xstart 和 xend之间的气球。你不知道气球的确切 y 坐标。

一支弓箭可以沿着 x 轴从不同点 完全垂直 地射出。在坐标 x 处射出一支箭，若有一个气球的直径的开始和结束坐标为 x``start，x``end， 且满足 xstart ≤ x ≤ x``end，则该气球会被 引爆 。可以射出的弓箭的数量 没有限制 。 弓箭一旦被射出之后，可以无限地前进。

给你一个数组 points ，返回引爆所有气球所必须射出的 最小 弓箭数 。

如何处理重叠的气球。首先要对气球进行排序，先按照左边界排序，再去看右边界。每一次重叠的时候，都要更新一下右边界，原因是要这个值是能够射两个气球的最大右边界值。

时间复杂度：$O(n)$，遍历一遍数组。

空间复杂度：$O(1)$，存放箭的数量。

13.无重叠区间

给定一个区间的集合 intervals ，其中 intervals[i] = [starti, endi] 。返回 需要移除区间的最小数量，使剩余区间互不重叠 。

这道题和上一道题的射箭很像，无重叠区间的个数就是箭的个数，所以可以复用，但是这个里面[1,2]和[2,3]是算两个区间的。

其他解法也可以按照元素进行排序，如按照第一个或者第二个元素排序，按左边界排序的话，就要对右边界的值进行更新，通过边界值将几个区间进行分割，得到独立的区间。

时间复杂度：$O(n)$，遍历一遍数组。

空间复杂度：$O(1)$，计数的数量。

14.划分子母区间

给你一个字符串 s 。我们要把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。

注意，划分结果需要满足：将所有划分结果按顺序连接，得到的字符串仍然是 s 。

返回一个表示每个字符串片段的长度的列表。

这个题有点抽象，主要是要搞明白题意，以及怎么使用hash表来记录当前的最远距离。同时，对于左边序号的更新也要注意。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(1)$，使用的hash数组是固定大小

15.合并区间

以数组 intervals 表示若干个区间的集合，其中单个区间为 intervals[i] = [starti, endi] 。请你合并所有重叠的区间，并返回 一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间 。

还是区间的问题，先排序再比较，基本思路不变。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(n)$，使用的数组是固定大小

16.单调递增的数字

当且仅当每个相邻位数上的数字 x 和 y 满足 x <= y 时，我们称这个整数是单调递增的。

给定一个整数 n ，返回 小于或等于 n 的最大数字，且数字呈 单调递增 。

使用暴力算法太笨了，如331->330明显不可以，所以可以直接跳过一些情况。331-329-229。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(1)$，使用一个int存储位数。

17.监控二叉树

给定一个二叉树，我们在树的节点上安装摄像头。

节点上的每个摄影头都可以监视其父对象、自身及其直接子对象。

计算监控树的所有节点所需的最小摄像头数量。

思想是从叶子结点向上，遇到没有覆盖的就在器父节点处添加摄像头。分为三种情况，无覆盖，有摄像头，有覆盖分别用012表示。

难点在于对情况的划分和对二叉树的访问方法。

一共三种状态的话，对于左右节点就是有$2^3=8$种情况。对这些情况分类讨论就行。
